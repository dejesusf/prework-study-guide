HTML

Elements

  • The 'HEAD' contains informtion about the webpage.
  • The 'BODY' represents the visible content shown to the user
  • The 'STYLE' defines the CSS styles associated with a website
  • The 'BASE' defines the base url for a webpage
  • The 'TITLE' defines the title of the webpage
  • 'LINK' connects an external resource to the HTML document.
  • 'META' defines metadata such as: character set, description, keywords, author, and viewport
  • 'SCRIPT' embeds client-side scripts in the HTML docoment
  • 'DIV' creates a division in the page
  • 'SPAN' creates in-line division
  • Header elements: 'h1', 'h2', 'h3', 'h4', 'h5', 'h6', go from large titles, to small subtitles. H1 should typically only be used once in a document.
  • 'UL' creates Unordered Lists
  • 'OL' creates Ordered Lists
  • 'IMG' contains info about images and can contain various attributes
  • 'BR' creates a line break (empty space) between two blocks of content
  • 'A' is an anchor element

Attributes

  • 'SRC' defines the location of an image
  • 'ID' identifies elements and selects them

CSS

  • Inline CSS: Used when a developer wants to style an element directly in the HTML file. This is useful for quick changes, but it is less flexible than external and internal styles.
  • External CSS: The most common way to stylesheet. When using an external stylesheet, we link the file using a LINK element placed in the HTML's HEAD.
  • Separation of concerns: widly used term in web development and refers to separating code into different files based on use.
  • Declarations: Contains two important components: CSS property and value of property.
  • Margin: Indicates how much space we want around the outside of the element. The margin is not visible and only adds a buffer.
  • Padding: Adds space around content issue inside an element. Increased padding = making the entrire element bigger.

GIT

GIT CHEATSHEET

Setup

  • git config --global user.name "[firstname lastname]": set a name that is identifiable for credit when review version history
  • git config --global user.email "[valid email]": set an email address that will be associated with each history marker
  • git config --global color.ui auto: set automatic command line coloring for GIT for easy reviewing

Stage & Snapshot

  • git status: show modified files in working directory, staged for your next commit
  • git add [file]: add a file as it looks not to your next commit (stage)
  • git reset [file]: unstage a file while retaining the changes in working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not yet committed
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot

JAVASCRIPT

  • Variable: A container that allows us to store data in the code. This is notated with the keyword VAR.
  • Control Flow: The order in which a computer executes code in a script. JavaScript runs from top to bottom.
  • Falsy: A value that is 0, -0, On, "", Null, undefined, NaN, or False.
  • Truthy: All other values, including True.
  • Array: A single variable that is used to hold a group of data. Typically used to hld data that is related in some way.
  • Function: Set of instructions that tells the computer how to perform a certain task.
  • Chrome DevTools allows developers to debug and test the code that is written.
